1. Executive summary

FinLens ranks United States FDIC-insured banks by their probability of financial distress or failure within four quarters, using only public quarterly Call Report financials. It is decision support for off-site monitoring and exam prioritization, not investment, deposit, or supervisory advice. The problem is framed as a discrete-time hazard on a per-bank-quarter panel: given a bank alive at quarter end, what is the probability it fails within the next four quarters. The served model is a calibrated, monotone-constrained gradient-boosted classifier scoring 0.301 PR-AUC on a held-out window of 66 real failures, with every claim reported against bootstrap intervals because at this rare-event rate point estimates are not, by themselves, a defensible result.

2. Problem framing

Bank failure is a time-to-event problem: a bank is observed over many quarters, its condition changes between filings, and at some point it may close. Modeling it as a single static classification throws away the time structure and the censoring. FinLens instead uses a discrete-time hazard: each bank-quarter is one observation, the target is conditional on survival to that quarter, and quarters are handled as an integer ordinal (obs_qord = year*4 + (quarter-1)) so gaps are exact arithmetic and the embargo logic can reason about them without date parsing. The default horizon is four quarters (one year); an eight-quarter horizon is also produced.

3. Data sources and the data ceiling

Three free public sources, no paid data:

  • FDIC BankFind Suite, /financials (banks.data.fdic.gov): quarterly per-CERT financial ratios and dollar fields, the feature source for the served model.
  • FFIEC Central Data Repository, bulk Call Reports (cdr.ffiec.gov): originally filed Call Report schedules, used for point-in-time integrity testing and for live forward scoring.
  • FDIC failures API (api.fdic.gov/banks/failures): true closures (RESTYPE FAILURE) give the failure dates that define the labels.

The panel is 2008Q1 to 2026Q1: 448,661 bank-quarters across 8,803 banks, with 2,138 labelable failure events and a base rate near 0.5 percent.

The binding constraint is data, not compute, and it is reality, not unfinished work. Only 66 real failures fall in the out-of-time test window. Extending history to 2001 was built and tested, and it REDUCED out-of-time PR-AUC from 0.219 to 0.139, because the early 2000s banking regime dilutes the recent-failure signal; it was reverted. The roughly 2,400 failures of the 1980s and 1990s Savings and Loan crisis are unreachable, because machine-readable Call Reports begin in 2001. Confidential supervisory data (CAMELS exam ratings, intraday liquidity, deposit flows) is not public at any price. So the model sits at its data ceiling.

4. Panel construction

The unit of observation is (CERT, quarter), where CERT is the FDIC certificate number and the quarter is a Call Report filing. Each row carries that quarter's financial ratios; a bank contributes one row per quarter it exists. The pipeline (ml/scripts/build_dataset.py) fetches financials, builds the panel, engineers features, attaches the four- and eight-quarter labels, and materializes an immutable DuckDB table ml.training_dataset. Identifiers are de-duplicated case-insensitively (DuckDB is case-insensitive, so a raw field ROA and a derived feature roa would otherwise collide).

5. Feature engineering (full column dictionary)

34 CAMELS-aligned features: levels (capital, asset quality, earnings, liquidity), trends (quarter-over-quarter and year-over-year deltas), and peer-relative z-scores within asset-size bands. Each carries an economically signed monotone constraint so the model cannot learn a perverse relationship a model-risk validator would reject. The sign convention: +1 means a higher value must not lower predicted risk; -1 means a higher value must not raise it; 0 is unconstrained.

A real feature bug, found and fixed. Noncurrent loans were initially built from FDIC P9LNLS, which is loans 90+ days past due AND STILL ACCRUING only (about 50 percent zero, which is economically normal because troubled loans move to nonaccrual). The correct field is NCLNLS, total noncurrent (nonaccrual plus 90+, about 11 percent zero). Correcting it lifted the served model from 0.221 to 0.301 and made noncurrent the number-two driver.

6. Labels and leakage control

For observation quarter q (ordinal), failure quarter f, and last observed quarter L, the rule (ml/finlens_ml/labels.py) is:

  • f exists and f ≤ q: drop (already failed).
  • f exists and q < f ≤ q+H: label 1 (fails within the horizon).
  • f exists and f > q+H: label 0 (survives the horizon).
  • no failure and L ≥ q+H: label 0 (observed alive through the horizon).
  • no failure and L < q+H: drop (right-censored: a merger or the end of data).

Survival presence in the panel is the ground truth, so healthy mergers are correctly censored and the last H quarters (whose outcome cannot be confirmed) are dropped, never labeled negative. An embargo equal to the horizon plus the reporting lag (additive) guarantees a training row's label window ends strictly before the test window begins; it is enforced at runtime by assert_no_temporal_overlap.

7. Train/test splits

The headline evaluation uses final_holdout_split: the last 28 quarters (about 2019 to 2026, a long window that contains the 2023 failure cluster) are the out-of-time test set, with the embargo applied so no training label peeks into it. Grouping is by CERT, so a bank never appears on both sides. For uncertainty about variance, rolling_origin_folds builds expanding-window out-of-time folds at successive origins and refits at each.

8. Model and hyperparameters

A LightGBM gradient-boosted classifier (discrete-time hazard) with monotone constraints, calibrated on a stratified in-training holdout. Every notable choice and why:

  • Gradient boosting over a linear model: captures nonlinear interactions among CAMELS ratios; the penalized logit is kept as the regulatory-style benchmark.
  • Monotone constraints: enforce economically signed relationships; they cost nothing measurable here (the constrained model matches the unconstrained GBM, 0.301 vs 0.270) while remaining examiner-legible.
  • Capped scale_pos_weight: the raw negative-to-positive ratio is so extreme the model saturates instantly; the weight is capped (a tuned spw_cap) so positives stay emphasized and calibration fixes the probabilities.
  • Tree count from out-of-time early stopping on average precision: AUC saturates at about one tree on so few positives, so average precision is used to let the model grow; the served model uses the discovered count, n_estimators = 38.
  • PR-AUC as the headline: at a sub-1-percent base rate, accuracy and ROC-AUC mislead; PR-AUC tracks how well true positives concentrate at the top of the ranking.
  • Hyperparameters tuned with Optuna over inner time-series CV folds (not the final holdout), scoring mean inner-fold PR-AUC, with a median pruner. Best parameters:

9. Calibration

Probabilities are calibrated on a stratified in-training holdout so the calibrator sees positives. A bake-off (ml/scripts/calibration_conformal.py) compared isotonic, Platt, and inductive Venn-Abers on a held fold: isotonic won (lowest ECE, about 0.00019) and was stable across bootstrap resamples. Per-instance Venn-Abers and split-conformal intervals were found vacuous at this base rate (most scores cluster near zero), so they are not shipped; that is reported honestly rather than presenting an uninformative interval.

11. Uncertainty and the G0 foundation

With only 66 positives, point estimates are reported with 95 percent bootstrap intervals: PR-AUC [0.191, 0.438], recall at 200 [0.419, 0.657]. A paired bootstrap confirms the edge over the logit is real: difference [0.070, 0.222], probability the model beats the logit 100.0 percent.

Minimum detectable effect. An external-truth simulation (G0, ml/scripts/g0_power_sim.py), which deliberately does not bootstrap the one held-out realization, shows the chosen interval method (bca) covers near nominal, and the paired gate's power to detect a 0.02 PR-AUC improvement is only about 6 percent. The honest conclusion: no tuning or ensembling change is statistically validatable on this test set, so model selection relies on inner rolling folds, and the headline is governed by the data, not by unfinished optimization.

12. Explainability (SHAP)

Global importance is the mean absolute SHAP value per feature over a bounded reservoir sample of out-of-time-era rows. Capital, asset quality, and earnings dominate, consistent with the bank-failure literature. Local per-bank SHAP reason codes are validator-facing transparency, not a legally sufficient adverse-action statement.

13. Effective challenge and ensembles

The model is benchmarked against a penalized logit and an unconstrained GBM, and seed-bagged and stacked ensembles are measured. Because every interval overlaps at this positive count, rungs are ranked by point estimate with that caveat stated, not treated as statistical separation.

14. Competing risks (failure vs merger)

Mergers are far more common than failures (Aalen-Johansen cumulative incidence 0.739 versus 0.185, about 4x). The model censors mergers; the informative-censoring bias this could introduce is quantified, not assumed: only 2.2 percent of merger-exit banks were elevated-distress at exit, so the downward recall bias is small. A discrete-time Fine-Gray subdistribution cross-check (ml/scripts/fine_gray.py) confirms the correction is immaterial to the served ranking.

15. Point-in-time data and forward scoring

Originally filed FFIEC Call Reports were pulled to test point-in-time feature integrity. They reconciled exactly for 2014 and later and surfaced the noncurrent field bug. As a training source the full point-in-time panel underperforms the restated panel (0.131 versus 0.175) because originally filed data is inherently noisier than restated, so it is not the training source. It is, however, the correct data path for live forward scoring: the current quarter has no restatement yet. The Early Warning surface offers a live forward score with explicit framing that it is a model estimate, not a forecast that a bank will fail, and that the base rate is under one percent.

16. Honest limitations

  • Rare events: about 66 out-of-time failures cap statistical power; this is the dominant ceiling.
  • Public data only: no confidential supervisory, intraday-liquidity, or deposit-flow data.
  • Originally filed data is noisier than restated, so point-in-time loses as a training source.
  • Pre-2001 history is unreachable; extending to 2001 was tested and hurt.
  • SHAP is transparency, not a legally sufficient adverse-action reason code.

17. Engineering, reproducibility, governance

The system is $0 (free public APIs only), reproducible (fixed seeds, pinned feature set, committed metrics), and gated: a CI metric gate blocks promotion of a degraded or leaky model (PR-AUC must beat the logit by a margin, OOT ROC must stay below a leakage ceiling, ECE within bound, paired-bootstrap edge significant). Serving resolves the MLflow champion alias with the pinned local artifact as offline fallback, loaded via skops with an explicit trusted-type allow-list (never raw pickle). The plan and the built result were each signed off by an adversarial three-reviewer committee (UI, ML, and banking domain). Aligned with the principles of SR 11-7 (non-binding here; this is a portfolio demonstration, not a regulated production model).
